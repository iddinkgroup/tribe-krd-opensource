Omschrijving

Om vanuit KRD webservices aan te kunnen roepen heeft KRD de adresgegevens van de webservice nodig. Daar de KRD applicatie in verschillende omgevingen draait zijn twee adressen noodzakelijk: één voor de productieomgeving, en één voor de overige omgevingen. Onderstaande gegevens zijn benodigd.

  • Endpoint address: de URL die naar de service die aangeroepen moet worden verwijst
  • Namespace URI: de namespace die de service in de WSDL identificeert
  • Service name: de naam van de service die aangeroepen moet worden

Voorbeeld:

Volgend XML voorbeeld geeft de definitie van een webservice aan:

<wsdl:definitions
    xmlns:eduarte="http://eduarte.topicus.nl/eduarte/SynchronisatieService"
    name="SynchronisatieService"
    targetNamespace="http://eduarte.topicus.nl/eduarte/SynchronisatieService">

    <!-- ... -->
    <wsdl:service name="SynchronisatieService">
        <wsdl:port binding="eduarte:SynchronisatieServiceSOAP" name="SynchronisatieService_Port">
            <soap:address location="https://www.example.com/endpoint" />
        </wsdl:port>
    </wsdl:service>
</wsdl:definitions>

Voor deze service is de endpoint address gelijk aan https://www.example.com/endpoint. De service name is SynchronisatieService en de namespace is http://eduarte.topicus.nl/eduarte/SynchronisatieService.
